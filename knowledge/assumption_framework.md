# Assumption Framework

Explicit reasoning about **what must happen** for a threat to beat you, **what could still happen**, and **what you can deprioritize in this frame**.

Used in teach-back, coach reveal, and litigation when a read fails.

---

## Three Tiers

| Tier | Definition | Coach question |
|------|------------|----------------|
| **Must happen** | Necessary conditions for the primary threat to beat you | "What *has* to be true for them to score this way?" |
| **Could happen** | Live branches you still respect | "What else can they do in the next 1–2 seconds?" |
| **Doesn't need to happen** | Ruled out or low priority *here* | "What are you ignoring — and why is that safe?" |

---

## Rules

1. Every **must** ties to a **visual cue** or is labeled **inferred**.
2. Every **doesn't need** includes **why_ruled_out** (cue-based).
3. Wrong **must** assumptions are the #1 teach-back litigation target — stronger than generic "missed threat."
4. **Could happen** branches should align with [situation_relationship_graph.md](situation_relationship_graph.md) `branches_to` edges.
5. When a read fails, identify which **must** was false or which **could** was underweighted.

## Examples by Situation

### 2-on-1 (odd_man_rush)

| Tier | Content |
|------|---------|
| Must happen (for cross-crease to beat delay) | Pass lane open; carrier can make tape-to-tape pass; middle drive available |
| Could happen | Shot from circle, deke, late pass as backchecker arrives |
| Doesn't need to happen | Weak-side wrap from poor angle; point shot |

### East-west seam

| Tier | Content |
|------|---------|
| Must happen (for one-timer to beat you) | Pass completes across slot; receiver shoots before you reset |
| Could happen | Carrier shoots, net-front tip, walk-out |
| Doesn't need to happen | Rush odd-man (different family) |

### Point shot + traffic

| Tier | Content |
|------|---------|
| Must happen (for tip to beat deep set) | Shot through traffic; deflection changes puck path |
| Could happen | Clean shot, miss net, rebound |
| Doesn't need to happen | Cross-ice seam (if puck is at point) |

## Link to Outcome Paths

Each `outcome_path` should map to one or more **could happen** branches. The **primary read** should address the highest-posterior **could** branch after assumptions are stated.

**flip_condition** = which **must** or **could** changed.

---

## Teach-Back

User fills must / could / doesn't need before coach reveal. Review scores:
- False musts
- Missing could branches
- Unjustified doesn't need
